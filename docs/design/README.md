## 🧭 設計原則

### 1. 段階的実装
複雑な機能は段階的に実装し、各フェーズで動作する基本機能を提供

### 2. 下位互換性
新機能追加時も既存の設定・使用方法を維持

### 3. 拡張性
将来の機能追加が容易な設計構造

### 4. パフォーマンス重視
WSL環境での軽快な動作を最優先

### 5. ユーザビリティ
セットアップが簡単で、直感的な操作性
